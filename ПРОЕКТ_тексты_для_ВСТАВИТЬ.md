## ВСТАВИТЬ: название темы / проекта

**Система мониторинга биосигналов и трекинга физических упражнений «BioSignal Monitor» (MedApp)**

---

## ВСТАВИТЬ: актуальность

Рост интереса к дистанционному здоровью и домашним тренировкам требует доступных средств объективного контроля состояния организма и техники выполнения упражнений. Классические носимые датчики и спортивные приложения часто разобщены: сигналы ЭМГ, ЭЭГ, пульса и кожно-гальванической реакции нужно собирать, обрабатывать и хранить в единой системе, а анализ движений — автоматизировать без дорогого оборудования. Разработка веб-платформы с приёмом данных от микроконтроллера ESP32, визуализацией в браузере и подсчётом повторений по видео снижает порог внедрения для образовательных и прототипных задач.

---

## ВСТАВИТЬ: цель работы

Целью является создание программного комплекса для приёма, обработки и отображения биосигналов (пульс, ЭМГ, ЭЭГ, показатель стресса по GSR), ведения истории измерений и сессий мониторинга, а также генерации персональных программ тренировок с помощью языковой модели (Groq API) и оценки выполнения упражнений по видеопотоку с камеры (MediaPipe Pose) в веб-браузере.

---

## ВСТАВИТЬ: задачи

1. Спроектировать протокол обмена между ESP32 и ПК (UDP, бинарный пакет 11 байт с контролем CRC).  
2. Реализовать сервер приёма и обработки сигналов (нормировка АЦП 0–4095, расчёт ЧСС, стресса, мышечной активности, показателя ЭЭГ) и передачу результатов в REST API.  
3. Разработать веб-клиент: авторизация, роли пользователей, дашборд с графиками (пульс, ЭМГ, ЭЭГ, стресс), запись сессий и сохранение истории в SQLite.  
4. Интегрировать генерацию планов тренировок через Groq (JSON-ответ, сохранение плана в БД).  
5. Реализовать трекер упражнений в браузере: захват видео, детекция скелета, подсчёт повторений по углам суставов, сохранение журнала.  
6. Обеспечить эмулятор сигналов ESP для тестирования без аппаратуры.

## ВСТАВИТЬ: объект исследования / разработки

Объектом является процесс мониторинга физиологических показателей и контроля выполнения упражнений в условиях взаимодействия встраиваемой системы сбора данных (ESP32), серверной части на Python (Flask) и клиентского веб-приложения (HTML/CSS/JavaScript).

---

## ВСТАВИТЬ: предмет исследования / разработки

Предметом являются методы приёма и фильтрации биосигналов, визуализация временных рядов на клиенте, организация REST API и базы данных, а также алгоритмы оценки фазы движения по ключевым точкам тела (MediaPipe) для подсчёта повторений.

---

## ВСТАВИТЬ: используемые технологии и средства

- Языки: Python 3, JavaScript, C++ (прошивка Arduino для ESP32).  
- Сервер: Flask, SQLite, библиотека `requests` для обращения к Groq API.  
- Клиент: одностраничное приложение (SPA) в одном `index.html`, Fetch API, Canvas для графиков.  
- Компьютерное зрение: MediaPipe Pose (CDN).  
- Сеть: UDP (датчик → ПК), HTTP JSON (мост → API), раздача статики отдельным процессом.  
- Средства разработки: редактор кода, браузер Google Chrome / Microsoft Edge, при необходимости — эмулятор `esp_emu.py`.

---

## ВСТАВИТЬ: структура программного комплекса

Комплекс включает: прошивку/код ESP32 (`esp.cpp`) с опросом АЦП и отправкой UDP; приёмник `esp_server.py`, преобразующий пакеты в показатели и вызывающий `POST /api/update_data`; REST-сервер `site_api.py` (пользователи, сессии, история датчиков, тренировки, цели); сервер выдачи интерфейса `site_server.py`; единый фронтенд `index.html`; модуль `bones_rec` (Python) для автономного трекинга; скрипт запуска `main.py`. База данных `users.db` хранит учётные записи, планы, логи упражнений и историю показателей датчиков.

---

## ВСТАВИТЬ: описание алгоритма обработки сигналов (кратко)

Сырые отсчёты четырёх каналов ограничиваются диапазоном 12-битного АЦП (0–4095). Для пульса используется выделение переменной составляющей и детектирование локальных максимумов с оценкой периода и ЧСС в уд/мин с ограничением по разумным пределам; при недостаточности пиков допускается оценка по уровню сигнала. Стресс и мышечная активность переводятся в шкалу 0–100 по нормированному ходу АЦП; для ЭЭГ формируется сглаженный показатель активности 0–100. Состояние («Норма», «Стресс», «Аритмия», «Перегрузка») выводится по пороговым правилам.

## ВСТАВИТЬ: описание веб-интерфейса

Интерфейс содержит страницы: главная (описание системы), регистрация и вход, дашборд мониторинга с карточками показателей и четырьмя графиками (пульс, ЭМГ, ЭЭГ, стресс), журнал событий и таблицу истории датчиков; страница тренировок с генерацией плана ИИ и списком упражнений; трекер с зеркалированным видео и HUD (повторения, фаза, таймер); профиль с данными пользователя, историей сессий мониторинга и блоком прогресса/целей. Запись в историю датчиков выполняется только при включённом режиме «Запись» на дашборде.

## ВСТАВИТЬ: практическая значимость

Разработанный комплекс может использоваться в учебных лабораториях для демонстрации цепочки «датчик — сеть — сервер — визуализация», для прототипирования телемедицинских и спортивных сценариев, а также как основа для дальнейшего расширения (дополнительные датчики, отчёты для врача, мобильное приложение).

## ВСТАВИТЬ: выводы

В ходе работы реализована связка ESP32 → UDP → Python-мост → Flask API → веб-клиент с графиками и учётом пользователей. Достигнута визуализация ключевых показателей, ведение истории в SQLite, генерация программ тренировок через Groq и подсчёт повторений упражнений по видео в браузере. Система модульна: приём данных, API и интерфейс разделены, что упрощает сопровождение и развитие проекта.
