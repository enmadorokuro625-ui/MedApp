# 📋 MedApp — Система мониторинга биосигналов и трекинга упражнений

## 📌 Назначение проекта

**MedApp** — это медицинская веб-платформа для мониторинга биоэлектрических сигналов человека в реальном времени и трекинга физических упражнений с помощью компьютерного зрения.

Система решает две ключевые задачи:

1. **Мониторинг здоровья** — приём, обработка и визуализация данных от аппаратных датчиков (ЭКГ/пульс, ЭМГ, ЭЭГ, КГР), подключённых через микроконтроллер ESP32. Все показатели записываются в базу данных и отображаются в реальном времени на веб-дашборде.

2. **Трекинг упражнений** — AI-генерация персональных программ тренировок (через LLM Groq API) и подсчёт повторений упражнений в браузере с помощью камеры и библиотеки MediaPipe Pose.

## 🔧 Описание модулей

### 🌐 Веб-интерфейс

#### `index.html`
Одностраничное приложение (SPA) со всеми страницами, стилями и логикой внутри одного файла.

**Страницы:**
| Страница | ID | Описание |
|---|---|---|
| Главная | `pg-home` | Лендинг с описанием системы, анимированная ЭКГ-кривая |
| Вход | `pg-login` | Форма авторизации |
| Регистрация | `pg-register` | Форма регистрации с проверкой пароля |
| Дашборд | `pg-dash` | Мониторинг в реальном времени: метрики ЧСС/ЭКГ/ЭМГ/ЭЭГ, графики, лог событий, таблица истории датчиков |
| Профиль | `pg-profile` | Настройки пользователя, история сессий, прогресс тренировок, цели |
| Тренировки | `pg-workout` | AI-генерация программ тренировок, список упражнений |
| Трекер | `pg-tracker` | Камера + MediaPipe: подсчёт повторений упражнения в реальном времени |

**Ключевые технологии фронтенда:**
- **MediaPipe Pose** (JS CDN) — детекция скелета и подсчёт упражнений через камеру
- Vanilla CSS с CSS-переменными (тёмная тема, glassmorphism)
- Canvas-графики для сигналов ЭКГ/ЭМГ/ЭЭГ
- Асинхронные запросы к REST API (`fetch`)

### 🖥 Серверная часть

#### `site_api.py` — REST API (Flask, порт 5000)

Основной бэкенд. Предоставляет все API-эндпоинты.

**Эндпоинты:**

| Метод | Путь | Описание |
|---|---|---|
| `POST` | `/api/register` | Регистрация пользователя |
| `POST` | `/api/login` | Авторизация |
| `POST` | `/api/profile` | Обновление профиля |
| `GET` | `/api/user/{username}` | Данные пользователя |
| `POST` | `/api/workout/generate` | Генерация плана тренировки через Groq AI |
| `GET` | `/api/workout/plans` | Список планов тренировок |
| `GET` | `/api/workout/logs` | Журнал выполненных упражнений |
| `POST` | `/api/workout/log` | Запись результата упражнения |
| `GET/POST/DELETE` | `/api/sessions` | CRUD сессий мониторинга |
| `POST` | `/api/update_data` | Приём данных от ESP32 (запись в БД) |
| `GET` | `/api/last_data` | Последние показатели датчиков |
| `GET` | `/api/sensor_history` | История показателей с пагинацией |
| `GET` | `/api/workout/stats` | Статистика тренировок |
| `POST` | `/api/goals` | Создание цели |
| `PUT` | `/api/goals/{id}` | Отметка достижения цели |

**Генерация тренировок (Groq AI):**
- Используется модель `llama-3.3-70b-versatile` через Groq API
- Запрос отправляется через прокси
- Система формирует промпт с доступными упражнениями и настройками сложности
- Ответ — JSON с программой из 4–6 упражнений

#### `site_server.py` — Фронтенд-сервер (Flask, порт 8080)

Простой статический сервер, раздаёт `index.html` и вспомогательные файлы.

#### `main.py` — Точка входа

Запускает три сервиса как отдельные процессы:
1. `site_api.py` (API на порту 5000)
2. `site_server.py` (фронтенд на порту 8080)
3. `esp_server.py` (UDP-мост для ESP32)

Останавливает все по `Ctrl+C`.

### 📡 Аппаратная часть (ESP32)

#### `esp.cpp` — Прошивка ESP32-S3

Файл содержит **две части**:

1. **Python-приёмник** (строки 1–141) — скрипт для отладки: слушает UDP-пакеты от ESP32, декодирует, вычисляет BPM/стресс/мышцы, отправляет статистику обратно на ESP.

2. **C++/Arduino прошивка** (строки 142–381) — код для микроконтроллера ESP32-S3:

| Компонент | Описание |
|---|---|
| **Датчики** | 4 аналоговых канала: пульс (PIN 1), ЭМГ (PIN 2), ЭЭГ (PIN 3), КГР/GSR (PIN 4) |
| **Дисплей** | OLED SSD1306 128×64, I²C (SDA=8, SCL=9) |
| **Сеть** | Wi-Fi → UDP-пакеты на порт 5005 (broadcast) |
| **Сэмплирование** | 50 Гц по аппаратному таймеру |
| **Протокол** | 11-байтный бинарный пакет: `[0xAA, P_H, P_L, M_H, M_L, E_H, E_L, G_H, G_L, CRC, 0x55]` |
| **Режимы дисплея** | ALL (4 графика), фокус на одном канале, MED_STATS (BPM/стресс/мышцы) |

**Вычисление медицинских показателей:**
- **BPM** — детектирование пиков пульсового сигнала по порогу (2800), расчёт интервала R-R
- **Стресс** — линейная нормализация сигнала КГР (0–100%)
- **Мышечная активность** — амплитуда ЭМГ за последние 50 сэмплов

#### `esp_server.py` — UDP→HTTP мост

Слушает UDP-пакеты от ESP32 на порту 5005, декодирует бинарный протокол, вычисляет:
- BPM (пиковый детектор с фильтрацией)
- Уровень стресса (из КГР)
- Мышечную активность (амплитуда ЭМГ)
- Альфа/бета ритмы ЭЭГ (через вариационный анализ)
- Состояние: «Норма», «Стресс», «Аритмия», «Перегрузка»

Отправляет JSON в `POST /api/update_data` ~10 раз/сек.

#### `esp_emu.py` — Эмулятор ESP32

Генерирует реалистичные синтетические данные датчиков для тестирования без физического устройства. Имитирует:
- Удары сердца (~75 BPM)
- Периодическое напряжение мышц
- Альфа-ритмы ЭЭГ
- Медленное изменение КГР

### 🦴 Модуль трекинга упражнений (`bones_rec/`)

Автономный Python-модуль для подсчёта повторений через камеру и MediaPipe.

> **Примечание:** Логика подсчёта упражнений также портирована в JavaScript внутри `index.html` для работы прямо в браузере.

#### `base_exercise.py` — Абстрактный базовый класс

Определяет общую логику для всех упражнений:
- **EMA-сглаживание** угла (α = 0.2)
- **Конечный автомат** подсчёта: `IDLE → UP → DOWN → UP (+1 повтор)`
- **Три уровня сложности**: easy, medium, hard (разные пороги углов)
- Абстрактные методы: `name`, `thresholds`, `get_landmarks()`, `draw_points()`

#### `exercise_tracker.py` — Трекер

Объединяет камеру, MediaPipe PoseLandmarker и объект упражнения:
1. Захват кадра с камеры (OpenCV)
2. Детекция позы (MediaPipe)
3. Передача ландмарков в упражнение → подсчёт
4. Отрисовка HUD (название, угол, повторения, фаза)

#### `exercises/` — Реализации упражнений

Каждый файл — наследник `BaseExercise`. Определяет:
- Какие 3 точки тела отслеживать (номера ландмарков MediaPipe)
- Пороговые углы для каждой сложности
- Визуализацию ключевых точек

| Упражнение | Файл | Точки (MediaPipe) | Угол |
|---|---|---|---|
| Отжимания | `push_ups.py` | 12→14→16 (плечо→локоть→запястье) | Угол локтя |
| Приседания | `squats.py` | 24→26→28 (бедро→колено→лодыжка) | Угол колена |
| Бицепс | `bicep_curls.py` | 12→14→16 | Угол локтя |
| Жим над головой | `shoulder_press.py` | 24→12→14 (бедро→плечо→локоть) | Угол плеча |
| Выпады | `lunges.py` | 24→26→28 | Угол колена |
| Джампинг-джек | `jumping_jacks.py` | 24→12→16 | Угол тела |
| Скручивания | `sit_ups.py` | 12→24→26 (плечо→бедро→колено) | Угол корпуса |
| Высокие колени | `high_knees.py` | 12→24→26 | Угол бедра |

#### `utils.py` — Утилиты

- `calculate_angle(a, b, c)` — вычисление угла ABC в градусах через скалярное произведение
- `get_point(landmark, w, h)` — преобразование нормализованных координат MediaPipe в пиксели

#### `m_pipe_catpure.py` — Захват камеры

Классы-обёртки:
- `CameraStream` — захват видеопотока через OpenCV
- `PoseDetector` — детекция позы через `mp.solutions.pose`

## 🗄 База данных (SQLite)

Файл: `users.db`

| Таблица | Описание |
|---|---|
| `users` | Пользователи: логин, email, ФИО, дата рождения, роль (Пациент/Врач), хеш пароля |
| `sessions` | Записанные сессии мониторинга |
| `workout_plans` | Сгенерированные AI программы тренировок (JSON) |
| `exercise_logs` | Журнал выполненных упражнений (повторения, время, сложность) |
| `user_goals` | Персональные цели пользователя |
| `sensor_history` | **Вся история показателей датчиков** (BPM, стресс, мышцы, альфа, бета, состояние, время) |

### Схема `sensor_history`

```sql
CREATE TABLE sensor_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL,
    bpm REAL DEFAULT 0,           -- Частота сердечных сокращений
    stress REAL DEFAULT 0,        -- Уровень стресса (0-100%)
    muscle REAL DEFAULT 0,        -- Мышечная активность (0-100%)
    alpha REAL DEFAULT 0,         -- Альфа-ритм ЭЭГ
    beta REAL DEFAULT 0,          -- Бета-ритм ЭЭГ
    state TEXT DEFAULT 'Ожидание', -- Норма / Стресс / Аритмия / Перегрузка
    recorded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

Запись происходит не чаще **1 раза в секунду** на пользователя (троттлинг в `update_data`).

## 🚀 Запуск

### Установка зависимостей

```bash
# Автоматическая (Windows)
setup.bat

# Ручная
pip install flask flask-cors requests
pip install mediapipe opencv-python numpy   # для bones_rec
```

### Запуск всех сервисов

```bash
python main.py
```

Или по отдельности:

```bash
python site_api.py       # API на порту 5000
python site_server.py    # Фронтенд на порту 8080
python esp_server.py test  # UDP-мост (для ESP32 или эмулятора)
python esp_emu.py        # Эмулятор датчиков (без железа)
```

Затем откройте **http://localhost:8080** в браузере.
